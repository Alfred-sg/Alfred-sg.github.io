JDBC

发表于 2019-11-20 | 分类于 java |

Java 数据库连接（JDBC）API 用于对接不同的数据库 —— 由遵守不同网络协议的数据库厂商提供第三方驱动程序，再由 Java 提供一个驱动管理器和一套 API，驱动程序就会注册到驱动管理器中，在此基础上，调用 API 即能访问驱动管理器，最后通过驱动程序与实际的数据库通信。驱动程序的实现经历过多次演进，目前是以纯 Java 语言的形式实现，能将 JDBC 请求直接翻译成数据库相关的协议。

JDBC 数据库 url 如 jdbc:mysql://127.0.0.1:3306/test?characterEncoding=UTF-8，需要描述主机名、端口号和数据库名等。使用 JDBC 的前置操作包含：下载驱动程序 jar 文件并注册（部分驱动不需要注册）；下载并启动特定数据库。完成前置操作后，就可以连接数据库并执行 sql。

api 接口包含：

  • Statement connection.createStatement(); 创建 Statement 对象，用于执行 sql。
  • void connection.close(); 关闭当前连接并释放 jdbc 资源。
  • ResultSet statement.executeQuery(String sql); 查询并返回 ResultSet 对象。
  • int statement.executeUpdate(String sql); 插入、更新、删除、DDL 建表等操作并返回受影响的行数。如果没有更新计数，则返回 0。
  • boolean statement.execute(String sql); 执行 sql，可能会产生多个结果集和更新计数。如果第一个执行结果是结果集，返回 true；反之为 false，即增删改操作。调用 getResultSet 或 getUpdateCount 可以得到第一个执行结果。
  • ResultSet statement.getResultSet(); 返回前一条查询语句的结果集，或返回 null。对每个结果集，该方法只能被调用一次。
  • boolean statement.getMoreResults(); 部分数据库允许一条 Select 查出多个结果集，getMoreResults 方法用于获取下一个结果集，如果存在，返回 true。
  • int statement.getUpdateCount(); 返回前一条更新语句影响的行数。如果前一条语句没有更新数据库，返回 -1。
  • void statement.close(); 关闭 statement 以及它的结果集。
  • boolean statement.isClosed; 是否关闭。
  • void statement.closeOnCompletion(); 当所有结果集被关闭后，自动关闭 statement。
  • boolean resultset.next(); 获取下一行数据，到达底部返回 false。
  • String resultset.getString(int columnNumber), String resultset.getString(String columnName), Int resultset.getInt(int columnNumber), Double resultset.getDouble(int columnNumber), Date resultset.getDate(int columnNumber), Blob resultset.getBlob(int columnNumber), Clob resultset.getClob(int columnNumber)… 获取指定列数据。
  • resultset.getObject(int columnNumber, Classtype), resultset.getObject(String columnLabel, Classtype); 以特定类型获取给定列数据。
  • int resultset.finalColumn(String columnName); 获取给定列的序号。
  • void resultset.close(); 关闭当前结果集。
  • boolean resultset.isClosed(); 判断是否关闭。
  • PreparedStatement connection.PrepareStatement(); 创建 PreparedStatement 对象，带 ? 的预备语句（使用场景如 where name = ?），便于重用 sql 操作语句。
  • void preparedStatement.setInt(int n, Int x)… 设置 PreparedStatement 对象第 n 个问号的值，即参数。
  • void preparedStatement.clearParameters(); 清除当前 PreparedStatement 对象的参数。
  • ResultSet preparedStatement.executeQuery(String sql); 查询。
  • int preparedStatement.executeUpdate(String sql); 插入、更新、删除、DDL 建表等操作并返回受影响的行数。如果没有更新计数，则返回 0。

jdbc 另外解决的问题包含：（日期格式等出入库时）转义；获取自动生成的主键；支持结果集可滚动查询上或下一条数据、结果集集成更新数据库操作（并非所有的数据库都支持可滚动、可更新操作，需要通过 DatabaseMetaData 接口的 supportsResultSetType, supportsResultSetConcurrency 判断）（可滚动、可更新操作会与数据库保持连接，消耗数据库资源）

注意：

  1. Connection, Statement, ResultSet 均因较大的数据结果占用不少内存资源以及数据库服务器上有限的资源，请求完成后须使用 try…catch…finally… 语句予以关闭。
  2. 尽量使用预编译并缓存的 PreparedStatement，并避免 sql 注入。
