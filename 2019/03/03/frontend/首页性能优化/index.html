首页性能优化

发表于 2019-03-03 | 分类于 前端 |

提升网页加载性能的手段无非合并请求、减小资源体积、缓存资源这几种。借助于前端工程化能力，我们能合并压缩 js, css 代码；制作 base64 图片，合成雪碧图等。此外，我们可以借助阿里矢量图标库制作字体图标；借助网上工具制作 WebP 图片。chrome 浏览器的 Page Speed 扩展工具能帮助我们查看网页加载性能，并给出有效的建议。

在前端编码层面，我们可以通过减少回流重绘、避免内存泄漏的方式提升网页加载性能。减少回流、重置的方法可参考网页渲染机制。关于内存泄漏，chrome 开发者工具 - more tools - Profile 面板可用于查看各函数的执行时间和 CPU 占用率。

在服务端领域，我们可以将静态资源放在 cdn 服务器上；使用 nginx 或 nodejs 搭建前置服务器，并基于 nginx 缓存、压缩静态资源。当使用 nodejs 制作前置服务器时，基于 pm2 或 egg-cluster 可开启多进程。

雅虎优化军规

  1. 减少 http 请求或优化资源：合并文件、使用雪碧图、使用 base64 图片、压缩混淆 js 脚本或 css 样式等
  2. 减少 dns 查询：减少网页资源主机名的数量（同时会影响可并行请求数量），建议使用 2 到 4 个主机名
  3. 避免重定向，重定向会一直等待到获取网页内容才渲染视图
  4. 使用 ajax 缓存：针对 get 请求，消息头添加 Expires，可以缓存响应
  5. 延迟加载：针对折叠的图片或者根据交互动作展示的内容等，可使用延迟加载的方式优化
  6. 预加载：在浏览器空闲的时候，预加载资源，包含跳转页面的资源、新版本的资源
  7. 减少 dom 节点的数量，可以提升 dom 树构建的速度，减少不必要的事件绑定
  8. 避免使用 iframe，iframe 会阻塞页面加载
  9. 杜绝 404，因其会占用服务端资源，浏览器还需要解析响应体
  10. 外链 css 样式和 js 脚本，可以减轻 html 体积，缓存资源。css 样式放在 head 中，可以逐步渲染页面。js 脚本放在底部，避免阻塞
  11. js 编码中避免回流、重绘；使用事件委托绑定事件，避免多次绑定
  12. 给 cookie 瘦身：cookie 尽可能小；设置合适的域级别和有效期
  13. 通过 Content-Encoding: gzip 响应头压缩资源
  14. 通过实体标签 ETags 响应头作为唯一标识，以判断浏览器缓存和服务端资源的更新状态，浏览器将传输 If-None-Match 查找资源
  15. 通过 Expires 响应头设置缓存失效时间，Cache-Control 条件性控制缓存
  16. 使用 CDN 内容分发网络
