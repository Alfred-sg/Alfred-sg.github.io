Servlet

发表于 2019-03-20 | 分类于 Spring Boot |

我们先来看一下，作为 Servlet 容器的 Tomcat 工作机制：

  1. 客户端首次发送请求，Tomcat 从磁盘中加载 Servlet 类并初始化执行 init 方法（依赖于 Servlet 配置）。
  2. Tomcat 将请求解析成 request，并将其转发给 Servlet。
  3. Servlet 调用 service 方法处理请求，获得 reponse 并发送回客户端。
  4. 再次发送请求，重复 2, 3 步。
  5. 当 Servlet 销毁时（卸载应用程序或关闭 Servlet 容器），调用 destory 方法。

Servlet 是运行在 web 服务器上用于接受和处理请求、获得响应的程序。我们可以通过继承抽象类 GenericServlet 处理遵循不同协议的请求，通过继承 HttpServlet 处理遵循 http 协议的请求。

ServletContext

当构建一个 web 应用时，针对不同的路由需要构建多个 Servlet 处理请求。和一般的容器技术相同，容器需要组织这多个 Servlet 的生命周期，并为这些 Servlet 注入常用的工具函数。ServletContext 上下文实例就作为Servlet 和容器通信的桥梁。在一个 web 应用中，ServletContext 实例只有一个。

Servlet API 开具了 ServletContext 接口，该接口会由不同的容器加以实现，并由容器在 web 应用启动时进行实例化，以便管理 Servlet。当 Servlet 初始化后，Servlet 可通过 getServletConfig 方法获得 ServletContext 上下文实例，并通过这个实例获得一些常用的工作函数，比如全局缓存。使用 ServletContext 共享全局缓存适用于单机环境；在分布式部署的环境下，每台虚拟机都会有一个 ServletContext 实例，因此共享资源需要借助数据库等外部存储。ServletContext 的常用操作如下：

  1. 获取资源或查看其媒体类型。
  2. 注册 servlet 并作管理。
  3. 注册 filter 并作管理。
  4. 缓存全局数据，在数据变更时触发 listener。
  5. 设置 session 超时时间等…

Servlet

通过 ServletContext 接口，我们可以感知到，由不同容器的 ServletContext 类可用于维护全量的 Servlet。通过 addServlet 方法就可以将 Servlet 实例添加到容器中。在容器中，直接可以访问的是 ServletRegistration 实例，该实例可用于设置 Servlet 的初始参数、处理请求的路由规则、以及将 Servlet 置为异步处理模式等。

如上文，Servlet 在容器中被实例化，再由容器管理 Servlet 三个生命周期方法调用的时机。这三个周期方法就是：

  • init(ServletConfig config) 初始化方法。通常当 Servlet 容器初次接受请求时，就会尝试实例化 Servlet 并将其放入容器中，这时就会调用 init 方法（如果将 Servlet 配置为 web 应用部署和启动时自动启动，那么 init 方法就会即时被调用）。init 方法执行成功后，ServletConfig 和 ServletContext 对象就可以被访问了。当 init 方法抛出 ServletException 错误，Servlet 将不会放入容器中，请求也得不到处理。
  • service(ServletRequest req, ServletResponse res) 处理请求以获取响应。因为 Servlet 容器会以多进程的方式运行，共享的资源、缓存需要及时作同步。
  • destroy() 当 Servlet 移出 Servlet 容器时将被调用，如杀死进程。实现 destory 方法一般用于销毁状态、及时同步资源或缓存。

ServletRequest

当容器接受到请求时，会将其封装为 ServletRequest 实例，然后将这个实例转发给 Servlet 加以处理。

ServletResponse

ServletResponse 类在容器中用于辅助生成响应：getOutputStream 方法可用于生成二进制数据；getWriter 方法可用于生成字符串数据。如果响应包含二进制及文本形式的混合数据，就需要借助 ServletOutputStream 类（getOutputStream 方法本身会创建 ServletOutputStream 实例）。
