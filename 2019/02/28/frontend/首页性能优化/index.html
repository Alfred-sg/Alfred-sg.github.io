首页性能优化

发表于 2019-02-28 | 分类于 前端 |

网页渲染过程

Navigation Timing

各事件说明：

  • navigationStart 前一份文档卸载完成的时间戳b。如果没有前一份文档，则返回 fetchStart
  • redirectStart 如果发生了 HTTP 重定向，且重定向和当前文档同域，则返回重定向的 fetchStart。否则，返回 0
  • unloadEventStart 前一份文档的 unload 事件触发时间
  • unloadEventEnd 前一份文档的 unload 事件执行完成时间
  • redirectEnd 如果发生了 HTTP 重定向，且重定向和当前文档同域，则返回最后一次重定向接受完数据的时间。否则，返回 0
  • fetchStart 浏览器发起资源请求时，如果有缓存，则返回读取缓存的开始时间。否则，返回获取资源的时间
  • domainLookupStart 查询 DNS 的开始时间。如果请求没有发起 DNS 请求，如 keep-alive，缓存等，则返回 fetchStart
  • domainLookupEnd 查询 DNS 的结束时间。如果没有发起 DNS 请求，同上
  • connectStart 开始建立 TCP 连接的时间。如果请求是 keep-alive，缓存等，则返回 domainLookupEnd
  • secureConnectionStart 如果是 https 请求，则返回握手时间。否则，返回 0
  • connectEnd 完成 TCP 连接的时间。如果是 keep-alive，缓存等，同 connectStart
  • requestStart 发起请求的时间，包含请求服务器资源、本地缓存等
  • responseStart 服务器或本地缓存开始响应的时间
  • responseEnd 服务器或本地缓存结束响应的时间
  • domLoading 文档对象 document 已创建，文档状态被置为 ‘loading’ 前的时间
  • domInteractive 文档状态被置为 ‘interactive’ 前的时间
  • domContentLoadedEventStart 文档对象触发 ‘DOMContentLoaded’ 事件前的时间
  • domContentLoadedEventEnd 文档对象触发 ‘DOMContentLoaded’ 事件后的时间
  • domComplete 文档状态被置为 ‘complete’ 前的时间
  • loadEventStart 文档的 load 事件触发时间，如没有则返回 0
  • loadEventEnd 文档的 load 事件执行完成时间，如没有则返回 0

各事件说明：

  • Queuing 请求排队中。在 HTTP 1 上，浏览器仅允许每个源拥有六个 TCP 连接；若请求的资源为低优先级资源，将让位给高优先级的样式或脚本资源
  • Stalled 请求等待发送所用的时间，包含代理协商所用的任何时间
  • DNS Lookup 执行 DNS 查询所用的时间。页面上的每一个新域都需要完整的往返才能执行 DNS 查询
  • Initial Connection 建立连接所用的时间，包括 TCP 握手/重试和协商 SSL 的时间
  • SSL 完成 SSL 握手所用的时间
  • Request Sent 发出网络请求所用的时间
  • Waiting(TTFB) 等待初始响应所用的时间，也称为至第一字节的响应时间
  • Content Download 接收响应数据所用的时间

performance 查看性能

首页加载时间：

  • DNS 寻址时间：domainLookupEnd - domainLookupStart
  • TCP 连接耗时：connectEnd - connectStart
  • request 请求耗时：responseEnd - requestStart
  • 白屏时间：responseStart - navigationStart
  • 解析 dom 树耗时：domComplete - domInteractive
  • domready 时间(用户可操作时间节点)：domContentLoadedEventEnd - navigationStart
  • onload 时间(总下载时间)：loadEventEnd - navigationStart

performance 方法：

  • performance.getEntries()：每个资源的请求时间
  • performance.now()：获取从 performance.timing.navigationStart 到当前时间的毫秒，可用于计算方法的执行时间
  • performance.mark()：给相应的视点做标记
  • performance.measure(name, startMark, endMark)：计算方法的执行时间
  • performance.getEntries(name)：获取资源的加载时间
