umi实现原理

发表于 2019-06-17 | 分类于 前端工程化 |

依赖

af-webpack

  • dev.js: 基于 webpack, webpack-dev-server, react-dev-utils 启动本地调试服务器，以构造完成的 webpack 选项作为首参。
  • build.js: 基于 webpack, react-dev-utils 进行打包，构造完成的 webpack 选项为其参数内容。
  • getConfig: 约定大于配置形式获取 webpack 选项。
  • getUserConfig/index.js: 提供工具函数解析用户配置，或者监听配置文件变更并重启编译流程。执行机制在 “配置更新” 小节加以阐述。
  • readRc.js: 基于 strip-json-comments 读取 json 配置文件，该 json 文件允许添加注释。
  • registerBabel.js: 基于 @babel/register 编译后续 require 加载的模块。

配置更新

当开发者更新配置文件时，脚手架需要重启编译流程。在 af-webpack 中，文件监听借助于 chokidar 库；重新编译借助于 node 进程间的通信，即当用户更改配置文件时，主进程发送消息，子进程接受消息后，重启编译流程。

babel-preset-umi

约定大于配置形式组合多个 babel-preset-, babel-plugin-。

eslint-config-umi

eslint-config-umi 基于 eslint-config-react-app 作代码校验。
