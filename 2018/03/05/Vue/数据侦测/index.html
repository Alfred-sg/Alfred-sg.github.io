数据侦测

发表于 2018-03-05 | 分类于 vue |

前言

数据侦测即监听对象属性或数组项的变动，触发后续的处理流程。通常，数据侦测基于观察者模式实现，当访问数据时，调用 Observer.subscribe(topic, listener) 方法，以绑定函数形式添加订阅者；当更新数据时，调用 Observer.publish(topic, …args) 方法，执行后续的处理流程。

上述代码存在的问题：

  1. 对于 reactiveDataManager.set(key, val) 执行绑定函数的代码段，可否简化为由 data[key] = val 形式的赋值语句启用后续的处理逻辑。
  2. 绑定函数显式指明，实际业务中，数据变更后的动作并不止于重绘视图，还包括计算属性更新(update computed value)、执行副作用函数(dispatch action)等。

对于问题1，可通过 Object.defineProperty(targer, key, descriptor) 方法修改访问器属性实现。

对于问题2，可通过将硬编码的 rerenderView 绑定函数替换为更灵活的订阅者管理器(SubscriberManager)实现。即在响应式数据变更过程中，实时查找依赖于这些响应式数据的订阅者，并触发订阅者的后续动作。

需要指明的是，当将响应式对象的单个属性视作为观察者模式中的主题对象时，单个观察者就可能依赖于多个主题对象(目标对象)，如计算属性 fullname 依赖于 firstname, secondname；而当其他属性变更时，也没必要重新计算 fullname。面对这种情况，可以通过 Observer.subscribe(reactiveData, key, listener) 方式声明绑定函数 listener 需要在 reactiveData[key] 数据变更时执行。显式指定监听属性的同时，也必然要显式移除，当 key 属性不存在时或者当响应式数据 reactiveData 销毁时。虽然可以将 listener 函数以 { bindTo: [ keyPath ], linstener } 形式注入到 reactiveData.listening 属性中，当数据变更时，再取出执行，诚如 backbone 的实现；但是，这样处理仍不够优雅。在 Vue 的实现中，listener 绑定函数以 reactiveData 为上下文，当其执行过程中，需要访问 this[key] 属性时，从而调用 descriptor.get 方法，因此，通过将 listener 存入某个闭包变量中，SubscriberManager 就可以感知当前需要添加的订阅者为谁。

响应式数据

在 Vue 中，index.js 模块提供的 Observer 类用于将对象或数组转化成响应式数据。若为对象，遍历其属性，由 defineReactive(target, key, value, customSetter?, shallow?) 调用 Object.defineProperty(obj, key, descriptor) 改写其访问器属性，get 方法更新订阅者，set 方法触发执行订阅者的绑定函数；若为数组，改写其 push 等原型方法，并将数组项转化为响应式数据。

介于 Object.defineProperty 方法只能对存在的属性起作用，Vue 中提供 set(target, key, value), del(target, key) 函数用于实现新增和删除属性同样需要触发订阅函数的场景。
