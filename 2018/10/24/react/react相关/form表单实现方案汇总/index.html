form表单实现方案汇总

发表于 2018-10-24 | 分类于 类库 |

前言

本文将汇总 Form 组件（react 框架）的实现，间断更新。

antd-form 组件

ant design 中的 Form 组件基于 rc-form 实现。首先，我将介绍 rc-form 的实现，其处理流程为：

  1. createFieldsStore，创建 FiledsStore 实例，存储表单项的元数据、表单数据、校验信息、校验状态等。

    • fields 属性，存储表单项的值、校验信息、校验状态、脏值标识等，可在初始化时设置，由 updateFields, setFields, getField 等提供交互接口。
      • 初始化 constructor(fields) 和 updateFields(fields) 方法执行阶段，只接受 Field 实例，为全量更新。
      • setFields(fields) 可混入新的字段，并更新字段的值。
      • resetFields(ns) 方法只输出空值，本身不改变 fields 中存储的数据。
      • getField(name) 获取 fields 中存储的数据并字段名。
      • getFieldMember(name, member) 方法基于 getField，用于辅助获取 fields[name] 成员，如 isFieldValidating, isFieldTouched 等。
    • fieldsMeta 属性，存储表单项的元数据，主要包含注入表单项组件的 props，特殊如 meta.validate = [{ rules, trigger? }] 校验规则，meta.hidden 区分隐藏项，meta.validate 为校验规则和触发事件，meta.initialValue 初始值，meta.normalize(value, prevValue, values) 转换字段的值。由 setFieldMeta(name, meta), getFieldMeta(name) 提供交互接口。
      • setFieldsInitialValue(initialValues) 方法在 fieldsMeta 中混入初始值，且校验 fieldsMeta 数据在初始值之前创建。
    • clearField(name) 方法用于清除 fieldsMeta, fields 中的数据。
    • 备注：fieldsMeta, fields 属性均未严格定义其所存储的数据格式。
    • 备注：表单数据赋值。
      • setFields(fields)，见上文。
      • setFieldsInitialValue(initialValues)，见上文。
    • 备注：表单数据获取。
      • getFieldValuePropValue(fieldMeta) 根据 fieldMeta 以键值对输出字段的值（包含初始值），可由 fieldMeta.valuePropName 提供键或经 fieldMeta.getValueProps 转换。
      • getValueFromFields(name, fields) 从 fields 中获取字段的值 value，或者获取 fieldsMeta[name] 中的初始值。
      • getFieldValue(name) 基于 getNestedField, getValueFromFields，获取匹配字段的值。
      • getFieldsValue(names) 基于 getNestedFields, getFieldValue 获取表单数据，不包含隐藏项。
      • getAllValues 根据 fieldsMeta 获取表单的全量数据，包含隐藏项。键值对形式，值就是字段的赋值数据。
      • getNotCollectedFields 剔除隐藏项后，获取 fields 尚未收集的字段数据。键值对形式，值为 {name, value, dirty} 对象。
      • getNestedAllFields 剔除隐藏项后，获取 fields 中未作收集或收集的字段数据。键值对形式，值为 {name, value, dirty, validating?, errors?, touched?} 对象。
    • 备注：表单字段名获取。字段名支持嵌套结构，通过 ‘.’ 或 ‘[‘, ‘]’ 实现对象属性、数组项的赋值。
      • getValidFieldsName 剔除隐藏项的字段名列表。
      • getAllFieldsName 全量字段名列表。
      • getValidFieldsFullName(maybePartialName) 获取所有以 maybePartialName 起始或完全等值的字段名，并剔除隐藏项。
      • isValidNestedFieldName 校验表单项的字段名不允许是另一个表单项字段名的成员。
    • 备注：错误文案等。
      • getFieldError(name) 基于 getNestedField, getFieldMember，获取校验文案。
      • getFieldsError(names) 基于 getNestedFields, getFieldError，获取校验文案。
      • isFieldValidating(name) 基于 getFieldMember，获取校验状态。
      • getFieldsError(names) 基于 getValidFieldsName, isFieldValidating，获取校验状态。
      • isFieldTouched(name) 基于 getFieldMember，获取 touched 状态。
      • isFieldsTouched(names) 基于 getValidFieldsName, isFieldTouched，获取 touched 状态。
    • 备注：辅助方法。
      • getNestedFields(names, getter) 遍历剔除隐藏项后的字段名列表，使用 getter 作进一步操作，返回键值对。
      • getNestedField(name, getter) 支持参数 name 值是嵌套字段名的头，遍历匹配的字段名列表，使用 getter 作进一步操作，返回键值对或数组。

  2. createBaseForm，为用户自定义 Form 组件提供 HOC 容器，注入采集、校验表单项数据的方法。

    • getInitialState 获取组件初始 state 阶段，创建 FieldsStore 实例，instances 缓存字段组件，cachedBind 缓存绑定函数，clearedFieldMetaCache 缓存待移除的字段数据，renderFields 缓存已渲染的字段组件，domFields 缓存字段组件的 dom 节点，并注入 getFieldsValue, getFieldValue, setFieldsInitialValue, getFieldsError, getFieldError, isFieldValidating, isFieldsValidating, isFieldsTouched, isFieldTouched 方法。
    • getFieldProps(name, usersFieldOption) 装饰字段组件的 props，并为 FieldsStore 实例收集元数据，如指定 ref 函数 this.saveRef，收集字段组件的实例；校验规则转换及获取；绑定函数封装为 onCollectValidate, onCollect 方法；元数据 fieldMeta 及表单数据 field 混入 props。该方法在字段组件实例化过程中执行，首先清理 clearedFieldMetaCache 缓存，最后记录 renderFields，并输出 props。
    • getFieldDecorator(name, fieldOption) 装饰字段组件，以获取字段组件实例的 ref 引用及 props，存储为 fieldMeta.ref, fieldMeta.originalProps，目的是在 BaseForm 内置的 saveRef 执行过程中调用字段组件的 ref 函数，以及在绑定函数执行过程中调用 props 方法；写入初始值。
      • 与 getFieldProps 的差别，在于 getFieldDecorator 支持在字段组件中设置绑定函数，支持写入初始值。
    • saveRef(name, _, component) 组件挂载、卸载、重绘阶段都会调用，根据参数 component 判断组件的渲染情况。要点是，重绘阶段会执行两次 ref 函数（移除在前，更新在后），getFieldProps 却只执行一次，这样就有可能会造成在移除过程中，缓存的 field, fieldMeta 数据丢失（同样在 ref 函数中清理缓存）。若 component 为空值，clearedFieldMetaCache[name] = { field, meta } 收集字段的元数据和表单数据，清除 FieldsStore 实例中的相关数据，清理 instances[name], cachedBind[name] 缓存；若 component 为组件实例，domFields[name] 缓存标记为真值，记录 instances[name] 字段组件实例，尝试使用 clearedFieldMetaCache 恢复 FieldsStore 实例中的相关数据。
    • 备注：辅助方法。
      • getCacheBind(name, action, fn) 以 cachedBind[name][action] = { fn, oriFn } 形式缓存 onCollect, onCollectValidate, saveRef 方法等，便于以 name 值进行查找。

核心逻辑点：

  1. 支持嵌套字段名。
  2. ref 引用。
