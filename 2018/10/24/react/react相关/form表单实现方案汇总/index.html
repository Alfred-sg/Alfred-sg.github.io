form表单实现方案汇总

发表于 2018-10-24 | 分类于 类库 |

前言

本文将汇总 Form 组件（react 框架）的实现，间断更新。

antd-form 组件

ant design 中的 Form 组件基于 rc-form 实现。首先，我将介绍 rc-form 的实现，其处理流程为：

  1. createFieldsStore，存储表单项的元数据、表单数据、校验信息、校验状态等。
    • fieldsMeta 属性，存储表单项的元数据，主要包含注入表单项组件的 props，特殊如 meta.validate 为校验规则和触发事件，meta.initialValue 初始值。由 setFieldMeta, getFieldMeta 提供交互接口。
    • fields 属性，存储表单项的值、校验信息、校验状态、脏值标识等，由 updateFields, setFields, getField 等提供交互接口。getFieldMember(name, member) 方法基于 getField，用于辅助获取 fields[name] 成员，如 isFieldValidating, isFieldTouched 等。
    • clearField(name) 方法用于清除 fieldsMeta, fields 中的数据。
    • 备注：fieldsMeta, fields 属性均未严格定义其所存储的数据格式。
    • 备注：通过 meta.hidden 属性区分是否 validField。getFieldsValue 获取表单数据时将剔除隐藏的表单项，getNestedAllFields 方法执行过程又将获取所有的表单项。
    • 备注：表单项字段名支持嵌套结构，通过 ‘.’ 或 ‘[‘, ‘]’ 实现对象属性、数组项的赋值。isValidNestedFieldName 方法用于校验表单项的字段名不允许是另一个表单项字段名的成员。getValidFieldsFullName(maybePartialName) 所有以 maybePartialName 起始或完全等值的字段名。
    • 备注：getNestedField(name, getter) 方法基于 getValidFieldsFullName 实现，遍历匹配的全量字段名，用于辅助获取成员的校验信息或赋值数据，如 getFieldValue, getFieldError。
    • 备注：getNestedFields(names, getter) 方法基于 getValidFieldsName 实现，与 getNestedField 不同的是，参数 names 只能是全名。同样用于辅助获取成员的校验信息或赋值数据，如 getFieldsValue, getFieldsError。
    • 备注：getAllFieldsName，getAllValues 将通过 fieldsMeta 获取所有表单项的字段名或赋值数据，包含隐藏项。
  2. createBaseForm，提供 Form 组件，采集并校验内部表单项的数据。
