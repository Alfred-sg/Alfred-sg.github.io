功能检测

功能检测有四种方式：

  • 检测全局对象（如 window、navigator）上是否有某属性，示例：geolocation 功能检测
  • 创建一个元素，检测它是否包含某属性，示例：canvas 功能检测
  • 创建一个元素，检测它是否包含某方法，示例：vedio format 功能检测
  • 创建一个元素，为它设置属性，检测这个属性值是否发生改变，示例：input type 功能检测

DETECTING HTML5 FEATURES 详细罗列了检测 html5 功能的方法。文中还指明：

  1. 借助 Modernizr，可以检测大部分 html5、css3 功能。
  2. ie8 以下可借助 Explorercanvas 使用 canvas 功能。
  3. 借助 Video for Everybody 可以在不支持 vedio 的浏览器中唤起 Apple QuickTime 或 Adobe Flash。
  4. 借助 GeoPosition.js 使用 geolocation 功能。
  5. HTML5 Cross-browser Polyfills 提供了更多兼容性垫片。

标签

DOCTYPE

DOCTYPE 与浏览器的怪癖模式、标准模式有关，详情可参考 Activating Browser Modes with Doctype、WHAT DOES IT ALL MEAN?。历史上，DOCTYPE 标签的 dtd 属性会指示浏览器使用不同的文档模式解析 html，html5 提供了 <!DOCTYPE html> 作标准化处理。对此，不支持 html5 的浏览器会降级使用 html4.01 等历史版本的兼容模式解析文档。

html

根元素 html 不再需要声明 xmlns 命名空间、xml:lang 属性，仅需使用 <html lang=”en”> 形式。

head

head 标签用于包含页面的元信息 meta、加载的资源 script、link 等。

html5 允许以 <meta charset=”utf-8” /> 指代 <meta http-equiv=”Content-Type” content=”text/html; charset=utf-8”>，声明字符串编码等。

以 <link rel=”stylesheet” href=”style-original.css” /> 指代 <link rel=”stylesheet” href=”style-original.css” type=”text/css” />。因为 css 外部资源的 type 值只有一种。rel 告诉 css 引擎这是一个文件资源。

以 <link rel=”alternate” type=”application/atom+xml” title=”My Weblog feed” href=”/feed/“ /> 告诉联合订阅源阅读器（如 Google Reader）本站点有更新的新闻提要。此处需要指定 type；RSS 或 Atom 媒体类型指本站点有更新的 atom 提要。

link 标签 rel 属性的其他用处有：

  • rel=”author” 指定页面作者。
  • rel=”start”, rel=”prev”, rel=”previous”, rel=”next” 指定上一页。下一页。
  • rel=”icon” 指定图标，即通过 <link rel=”shortcut icon” href=”/favicon.ico”>。
  • rel=”license” 指版权许可条款。
  • rel=”nofollow” 指连接没有获得站点原作者认可。
  • rel=”noreferrer” 指链接不会泄露任何引用信息。
  • rel=“prefetch” 指预先获取资源，仅 firefox 支持。
  • rel=”search” 指引用的 OpenSearch 文档提供了一个专门用于搜索文档及其相关资源的接口。
  • rel=”tag” 指标签适用于当前文档。

语义标签

  • section 指文档中的章节，通常包含一组内容和标题。
  • nav 指导航链接，通常包含一组链接。
  • article 指独立的内容，可以是帖子、文章、网络日志条目、用户评论、交互小部件等。
  • aside 指独立于主内容区域外的内容，如侧边栏。
  • hgroup 指章节中的标题组，如包裹一组 h1、h2 标题等。
  • header 指文档的头部区域，可能包含 h1、hgroup 等。
  • footer 指文档的尾部区域，包含文档作者、著作权信息、链接的使用条款、联系信息等。
  • time 指日期或 24 小时时间。
  • mark 指标记或突出显示文本。

兼容性问题

对于不支持 html5 的浏览器，html5 提供的语义标签就是 unknown element。我们首先需要知道浏览器会怎么处理元素，第一元素有其自由的样式，如 p、ul 节点；第二元素有层级关系，如两个未闭合的 p 元素会以兄弟节点显示，未闭合的 p 元素与 span 元素并列，span 将作为 p 的子节点。对于 unknown element，浏览器的处理策略是不对该元素设置样式，其样式属性来自于继承；层级关系上均作为兄弟节点处理。在不支持 html5 的浏览器中，unknown element 会使用内联样式；支持 html5 的浏览器则使用块状样式。因此在旧浏览器中，需要显式重置样式，参见 HTML5 Reset Stylesheet。IE 9 以下不会对 unknown element 设置样式，解决方案看这里?redirectedfrom=MSDN#_HTML_Parsing)。同时，IE 9 以下会将 unknown element 作为空元素对待，渲染时将其子节点作为 unknown element 的兄弟节点。处理方法是通过 createElement 创建元素，如下：

canvas

canvas 画布的初始用法如上，它提供的调用接口包含：

  • context.fillStyle：设置颜色、渐变等
  • context.fillRect(x, y, width, height)：使用 fillStyle 绘制矩形区域
  • context.strokeStyle：边框样式
  • context.strokeRect(x, y, width, height)：使用 strokeStyle 绘制边框
  • context.clearRect(x, y, width, height)：清除矩形；设置 canvas 画布的宽度也可以重置内容

canvas 画布是一个以左上角为坐标原点的二维坐标平面：

  • context.moveTo(x, y)：定义线条的起始坐标
  • context.lineTo(x, y)：定义线条的结束坐标
  • arc(x,y,r,start,stop)：定义原型
  • context.stroke：绘制线条
  • context.font：定义字体
  • context.textAlign：定义文本对齐方式
  • context.textBaseline：定义文本的基线
  • context.fillText(text,x,y)：绘制实心文本
  • context.strokeText(text,x,y)：绘制空心文本
  • context.createLinearGradient(x,y,x1,y1)：创建线条渐变
  • context.createRadialGradient(x,y,r,x1,y1,r1)：创建一个径向/圆渐变
  • context.drawImage(image,x,y)：绘制图像
  • context.drawImage(image, x,y,dw,dh)：缩放
  • context.drawImage(image,x,y,sw,sh,dx,dy,dw,dh)：裁剪

vedio

video 支持的常见视频格式（视频格式包含视频编码和音频编码格式）有：

  • MP4：带有 H.264 视频编码和 AAC 音频编码的 MPEG 4 文件，扩展名为 .mp4、.m4v
  • WebM：带有 VP8 视频编码和 Vorbis 音频编码的 WebM 文件，扩展名为 .webm
  • Ogg：带有 Theora 视频编码和 Vorbis 音频编码的 Ogg 文件，扩展名为 .ogv

常用接口为：

  • play：播放
  • pause：暂停

Geolocation

position 包含属性有：

  • coords.latitude：十进制数的纬度
  • coords.longitude：十进制数的经度
  • coords.accuracy：位置精度
  • coords.altitude：海拔，海平面以上以米计
  • coords.altitudeAccuracy：位置的海拔精度
  • coords.heading：方向，从正北开始以度计
  • coords.speed：速度，以米/每秒计
  • timestamp：响应的日期/时间

localStorage

cookies 最大存储容量为 4kb，且需要与服务端交互，影响传输速度。因此浏览器提供了 localStorage、sessionStorage，sessionStorage 只在当前会话期间有效。两者包含相同的接口：

  • setItem(key,value)：存数据
  • getItem(key)：取数据
  • deleteItem(key)：删数据
  • clear：清空
  • key(index)：得到某个索引的 key

if (Modernizr.localstorage) {
// 第一种方式
var foo = localStorage.getItem(“bar”);
// …
localStorage.setItem(“bar”, foo);

// 第二种方式
var foo = localStorage[“bar”];
// …
localStorage[“bar”] = foo;
} else {
// no native support for HTML5 storage :(
// maybe try dojox.storage or a third-party solution
}

history

history 以堆栈形式存储历史访问记录。

  • history.state: 获取栈顶——历史堆栈入口的状态值。
  • history.back(): 前往上一页，浏览器的回退按钮可以模拟此方法。
  • history.forward(): 前往下一页，浏览器的前进按钮可以模拟此方法。
  • history.go(num): 前往历史堆栈中的指定页面。
  • history.pushState(state, title, url): 将数据压入历史堆栈中。state 数据量须小于 640kb，firefox 会将其保存到本地磁盘中。firefox 也会忽略 title 参数。这一过程将触发 popstate 事件，但不会触发 hashchange 事件。
  • history.replaceState(state, title, url): 替换历史堆栈入口的数据。

popstate 事件，在历史堆栈入口数据变更时均会触发，包含调用 history.back, history.forward, history.go 方法，以及点击前进、回退按钮。但是 history.pushState, history.replaceState 方法不会触发 popstate 事件。

除了 popstate 事件，当 hash 路径变更，将会触发 hashchange 事件。
