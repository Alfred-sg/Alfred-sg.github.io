babel

2020-08-22
analyst, js

原理

babel 处理三步骤：parse 解析、transform 转换、generate 生成。

  • 解析过程由 babel-parser 模块实现，分为词法分析、语法分析两步。词法分析将源程序解析成令牌流，令牌的 type 包含一组属性描述该令牌；语法分析将令牌流转换成 AST。
  • 转换过程将遍历 AST，对节点进行添加、更新及移除等操作。此时插件将介入工作。
  • 生成过程将深度遍历 AST，将其转换成字符串形式的代码，并生成 source maps 源码映射。

解析

@babel/parser 源于 acorn。acorn 为插件架构，因此 babel-parse 同样为插件架构，以便支持 es、flow、jsx、ts 语法。@babel/parser 会产出 AST 抽象语法树（Babel AST format）。Babel AST format 基于 ESTree 规范；对于 jsx 语法，则基于 Facebook JSX AST。AST Explorer 直观地呈现了源程序会转变成怎样 AST 语法树及相应的 json 表现。AST 节点模型及语法节点为：

@babel/parser 插件机制基于 EstreeParser 子类扩展 Parser 父类实现。

@babel/parser 的处理流程为：

  1. 首先通过 Tokenizer 将源程序字符串解析成令牌流
  2. 其次通过 Parser 构建语法树

令牌指 Token 实例，它包含 type 属性指类型（如 num、string 等）、value 属性指具体的数值或字符串以及一些位置信息。令牌流的解析流程为：

  1. 首先通过 state#pos 标记当前处理的字符流索引
  2. 通过 charcodes 判断读取首个字符，根据首个字符的类型进行后续处理。比如读取的首个字符为 ‘.’，通过预读下一个字符判断是否为数字或 ‘…’ 或作纯 ‘.’ 处理。详见 tokenizer#getTokenFromCode 方法
  3. 根据字符流类型创建 Token 令牌，令牌的信息其一可以通过 state.type, state.value 存储，其二可以通过 state.tokens 存储。

Tokenizer 对外提供了逐个取词的能力。它暴露的接口有：

  • next：解析令牌流，存储在 state.tokens 中。
  • nextToken：解析一个令牌，信息存储在 state.type, state.value 中。
  • eat(type)：解析一个令牌。若令牌类型与 type 匹配，返回 true；否则返回 false。

单以 if (expr) stmt 语句为例，按上下文无关文法，它最终会形成 IfStatement 节点，以及其下的 SequenceExpression、Statement 节点。语义分析最终施诸于程序实现，因此 @babel/parser 实现了 parseIfStatement、parseExpression、parseStatement 方法。对于每种类型的 Node 节点，其构建过程为，@babel/parser 首先会使用 startNode、startNodeAt 等方法创建 Node 实例并其初始位置信息，然后在 parseXXX 过程中为该 Node 实例添加属性，如 IfStatement#test 条件语句、IfStatement#consequent 为执行体，最后调用 finishNode 方法为 Node 实例设置类型和结束位置信息。在此过程中，因为 Node 节点的嵌套层级，parseXXX 方法会呈现递归调用的特点；语句 Statement 也有多态特征，parseStatement 等方法中也有多个条件语句分支，其中一个是 parseIfStatement。与此同时，一些辅助性的标识如 ‘(‘、’)’ 不构成节点，会在程序中被剔除。

路径

为了新增、删除、更新节点，babel 使用 Paths 路径操纵节点。路径是一个节点在树中的位置以及节点信息的响应式 Reactive 表示。 访问者实际获得的参数是路径，而非节点。

模块

babel-traverse

babel-traverse 该模块用于遍历和更新节点。

babel-types

babel-types 适用于 AST 操作的 lodash 式工具库。功能包含类型检查、定义节点类型等。

babel-generator

babel-generator 根据 AST 生成代码和源码映射。

babel-template

babel-template 手动以字符串编写 AST。
