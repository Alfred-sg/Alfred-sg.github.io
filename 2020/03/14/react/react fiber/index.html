react fiber

发表于 2020-03-14 | 分类于 react |

前情荟萃

The crux of the change is transitioning from processing updates in a synchronous, recursive way to relying on asynchronous scheduling and prioritization of interface changes.

A look inside React Fiber 指明，react 演进出 fiber 架构的关键点在于渲染策略的变更，从原先的 Stack Reconciler 同步递归模式转变为 Fiber Reconciler 基于优先级的异步调度模式。这样就能避免其他文章所说的，Stack Reconciler 模式中的 JS 运算、页面布局和页面绘制将长期占用主线程，页面出现掉帧的现象。深入理解 React Fiber 引用了 Andrew Clark’s 的笔记，指出 Fiber Reconciler 包含如下特性：

  • 为不同类型的任务赋予优先级
  • 任务的暂停与恢复
  • 如果任务不再需要，可以中止
  • 复用之前已完成的工作

React Fiber 原理介绍 说明了 Reconciler 在 React 中的层次结构：

  • Virtual DOM 层：描述页面的元素构成，即 render 函数返回的结果。[译]深入React fiber架构及源码 坚持认为 Virtual DOM 实际上指的是不可改变的 React 元素树。
  • Reconciler 层：执行 reconciliation 流程，通过 diff 算法等将组件状态变化反应为视图内容，并触发 side-effects（调用生命周期方法、更新 ref）等。
  • Renderer 层：根据不同的平台，渲染出真实的内容，比较常见的是 ReactDOM 和 ReactNative。

推想 [译]深入React fiber架构及源码 提及的，React 渲染期间各种数据结构的流向比如：用户侧创建 Class 或 Function 组件、Host组件（如 DOM 节点）、Protals 等；通过 React.createElement 创建 React 元素树；基于 React 元素树创建 fiber 节点树（也称为 internal instances 内部实例树）；最后由 fiber 节点树获得具体平台相关的 public instance（也称为 Host instance，如真实的 DOM 树）。

fiber 节点树不像剥离 diff 算法后的 Virtual DOM 只是一种表现，其内部还集成了操作树的算法。[译]深入React fiber架构及源码 中称：

fiber 作为一种数据结构，用于代表某些 worker，换句话说，就是一个 work 单元，Fiber 架构提供了一种可跟踪、调度、暂停和中止 work 的便捷方式。

首次渲染时，fiber 节点基于 React 元素创建（createFiberFromTypeAndProps 执行期间）；更新时，fiber 节点将被重用，只更新部分属性，或者伴随着 React 元素的移除而销毁（基于 key 键）。

渲染完成的 fiber 节点树称为 current tree。当更新时，current tree 会维持不变，对于每个 fiber 节点，会依据 render 返回的 React 元素创建一个备用的 alternate fiber 节点。这些节点构成 workInProgress tree。当更新相关流程都走完后，workInProgress tree 会被刷新到页面上，并成为新的 current tree。

按 [译]深入React fiber架构及源码 描绘的，side effects 是组件 render 以外的流程。纯 render 可视为仅消费 state、props 获取 UI 的函数。其他如订阅 state 手动修改 DOM 节点、调用生命周期函数都视为 side effects。具体到不同类型的 fiber，side effects 可能包含：Host 组件（如 DOM 元素）的增删改节点操作；class 组件的更新 ref、调用生命周期。官方对 side effects 的描述为：

You’ve likely performed data fetching, subscriptions, or manually changing the DOM from React components before. We call these operations “side effects” (or “effects” for short) because they can affect other components and can’t be done during rendering.

[译]深入React fiber架构及源码 指出，React 渲染分为 render 阶段、commit 阶段。

render 阶段在首次渲染时生成 current tree；再次渲染时基于更新后的 state 值生成部分节点带有 side effects 标识的 workInProgress tree（render 阶段执行完成后也称为 finishedWork tree）。带有 side effects 标识的 fiber 节点会构成 effects list 线性列表，前一个 fiber 节点的 nextEffect 指向后一个 fiber 节点。render 阶段可以异步执行。

render 阶段使用 DFS 深度优先搜索算法遍历 fiber 节点树。在源码中由以下函数构成：

  • workLoopSync、workLoopConcurrent：使用 while 语句遍历 fiber 节点树（workInProgress tree）。
  • performUnitOfWork：while
  • beginWork：对当前 fiber 节点做处理，并将 fiber.child 返回到上游。
  • completeUnitOfWork：while
  • completeWork：while

render 阶段执行的生命周期方法有：

  • [UNSAFE_]componentWillMount (deprecated)
  • [UNSAFE_]componentWillReceiveProps (deprecated)
  • getDerivedStateFromProps
  • shouldComponentUpdate
  • [UNSAFE_]componentWillUpdate (deprecated)
  • render
