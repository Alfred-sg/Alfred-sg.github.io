react fiber hooks钩子篇

发表于 2020-03-10 | 分类于 react |

在 hooks 机制上，fiber reconciler 同样抽象了几种数据类型以承载状态变更的逻辑：Update、UpdateQueue、Hook、Effect 等。以 useState 说明其实现流程如下：

  1. 使用 renderWithHooks 封装组件渲染函数，每次执行期间重置 workInProgressFiber.memoizedState 属性（memoizedState 以链表的形式存储钩子）
  2. 渲染函数内部调用 useState，走 mountState 流程，借助 mountWorkInProgressHook 函数创建 hook 实例，并添加到 memoizedState 队列中，构造 dispatch 方法，以数组形式返回 hook.memoizedState 初始状态、dispatch
  3. 消费 state 进行渲染，workInProgressFiber 转变为 currentFiber（因此 currentFiber.memoizedState 中存储着组件渲染函数所使用的全量钩子），结束 mount 流程
  4. 再次渲染时，执行组件渲染函数将会重置 workInProgressFiber.memoizedState，而 currentFiber.memoizedState 依旧存储着全量钩子，因此挂载和更新流程可基于 currentFiber.memoizedState 判断。renderWithHooks 内部，正是借此使用不同的 ReactCurrentDispatcher，比如挂载时用 HooksDispatcherOnMount、更新时用 HooksDispatcherOnUpdate
  5. 基于 ReactCurrentDispatcher 的不同，重绘期间调用 useState，走 updateState 流程，以此完成状态变更
  6. 消费 state 进行渲染，结束 update 流程

hooks 中的状态变更与 redux 相仿，即采用 reducer(prevState, action) 计算 nextState。
