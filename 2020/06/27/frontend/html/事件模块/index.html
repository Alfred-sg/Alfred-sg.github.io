事件模块

2020-06-27
analyst, js

标准的现代浏览器为事件提供了 addEventListener、removeEventListener 等接口。

addEventListener 将事件监听器注册到 EventTarget 对象上，而 Node 接口继承了 EventTarget。事件监听器的 this 指向触发事件的元素引用。addEventListener 可以通过 capture 选项设置是否在捕获阶段执行（默认在冒泡阶段执行）；通过 once 选项设置只执行一次；通过 passive 选项阻止调用 preventDefault 函数。事件流机制可以参考 DOM3 文档。

在IE 9之前，需要使用 attachEvent，此时 this 的值会指向 window 对象。IE 8可以通过添加 polyfill 使浏览器支持 addEventListener、preventDefault、stopPropagation，但是IE 7及以下不支持 Element.prototype，也即添加 polyfill 也是无效的。polyfill 的实现可以参看 addEventListener。

除了浏览器支持的事件外，事件系统还支持自定义事件（称为 synthetic events 合成事件）。创建和触发 events 说明了创建、分发自定义事件的方法。

事件处理的复杂性在于事件的多样性，可参看 Event 接口。

jQuery

jQuery 使用 jQuery.Event 模拟事件对象，一方面可以解决 preventDefault、stopPropagation、stopImmediatePropagation 等方法的兼容性问题，另一方面可以通过 which 获取按键或鼠标按键类型 keyCode。

除模拟了事件对象之外，jQuery 还模拟了事件注册、移除、分发、事件冒泡、事件委托的整套机制。包含：

  • jQuery.event.add：注册事件。
  • jQuery.event.remove：移除事件。
  • jQuery.event.dispatch：触发事件。
  • jQuery.event.handlers：查找事件监听器。

其手法是：

  1. 通过缓存在元素上的 events 属性记录事件类型、绑定函数、命名空间、传递数据、实际触发事件的节点（在事件委托机制下，绑定函数会被记录到委托元素上）、guid 等信息。该属性的数据结构为数组对象，以时间类型为 key，绑定函数以数组形式作为 value。在绑定函数数组，委托事件监听器将处于数组的前列。
  2. 所谓事件注册，就是向 events 缓存数据中添加事件监听器；所谓事件移除，就是移除 events 缓存数据中的事件监听器。当然，为了使用户的交互行为触发事件，jQuery 会使用 addEventListener 接口为元素绑定 jQuery 内部的事件分发函数 jQuery.event.dispatch。
  3. jQuery.event.dispatch 处理机制，先找到元素该事件的绑定函数，然后通过是否有委托，向上查找所有组件节点委托的绑定函数（设定的触发节点需要包含元素）。构成两维数组形式，存储着每个节点的绑定函数数组。通过双层遍历执行绑定函数，若有 stopPropagation 或 stopImmediatePropagation 行为，则中止遍历。

特殊的，jQuery 通过 jQuery.event.special[eventType] 为特殊的事件类型设置钩子、转换的事件名等。详情可参看 event 模块。

在这套机制的基础上，jQuery 封装了 $.on、$.one、$.off 方法。

如上所述，jQuery.event.dispatch 只能触发 jQuery 模拟事件，因此在 $.trigger 方法的实现上，jQuery 既需要触发模拟事件，又需要触发原生事件，以及冒泡机制。详情可参看 trigger 模块。

react

简要的说明，react 首先使用 SyntheticEvent 等模拟事件对象。在事件触发时，SimpleEventPlugin 等插件会创建或从事件池中获取模拟事件对象实例，并将组件上的绑定函数存入实例属性中，react 内部实现的事件分发过程将调用绑定函数去处理模拟事件对象。为了让真实 DOM 也能触发 react 内部的事件分发机制，react 会将 dispatchEvent 方法绑定到 document 节点上。

模拟事件

react 同样模拟了事件对象，即 SyntheticEvent。它同样实现了 preventDefault、stopPropagation 方法，nativeEvent 属性指向原生事件，_targetInst 属性指向组件实例。后端技术中线程池，react 的事件系统有事件池，即 SyntheticEvent.eventPool。通过 SyntheticEvent.getPooled 从事件池中取出 SyntheticEvent 实例并重新执行初始化函数，或者当事件池为空时，创建 SyntheticEvent 实例。通过 SyntheticEvent.releasePooledEvent 清空 SyntheticEvent 实例的初始化数据、事件监听器等，重新装填到事件池中，以便下次取用。SyntheticEvent 实例的 _dispatchListeners 会置为绑定函数数组，_dispatchInstances 会置为触发组件实例。以下是事件池的实现源码：

SyntheticEvent 有以下子类，与原生 Event 对象 对应：

  • ResponderSyntheticEvent：包含 touchHistory 属性
  • SyntheticUIEvent：继承 SyntheticEvent，包含 detail 属性指点击数，view 指事件关联的 window 对象
  • SyntheticInputEvent：继承 SyntheticEvent，包含 data 属性指当前输入的字符串
  • SyntheticCompositionEvent：继承 SyntheticEvent，包含 data 指触发事件的输入方法所产生的字符。参考 CompositionEvent
  • SyntheticClipboardEvent：继承 SyntheticEvent，包含 clipboardData 属性指赋值的内容
  • SyntheticTransitionEvent：继承 SyntheticEvent，包含 propertyName 指 css 属性名；elapsedTime 指动效时长；pseudoElement 指伪元素
  • SyntheticAnimationEvent：继承 SyntheticEvent，包含 animationName 指 css 动画名；elapsedTime 指动画时长；pseudoElement 指伪元素
  • SyntheticFocusEvent：继承 SyntheticUIEvent，包含 relatedTarget 属性指通过 tab 按键失去焦点的元素
  • SyntheticKeyboardEvent：继承 SyntheticUIEvent，包含 location 按键位置、repeat、locale、ctrlKey、shiftKey、altKey、metaKey、getModifierState 判断 ctrl,shift,alt,meta 是否被按下、key 获取物理按键名、charCode 获取 keyPress 事件按键数值、keyCode 获取 KeyDown/Up 事件按键数值、which 获取按键数值
  • SyntheticMouseEvent：继承 SyntheticUIEvent，包含 screenX 鼠标指针相对于屏幕的 x 坐标、screenY、clientX 点击元素的 x 坐标、clientY、pageX 鼠标指针相对于文档的 x 坐标、pageY、ctrlKey、shiftKey、altKey、metaKey、getModifierState、button 鼠标按键、buttons 多个鼠标按键、relatedTarget、movementX 指 mousemove 事件发生的 x 坐标、movementY
  • SyntheticTouchEvent：继承 SyntheticUIEvent，包含 touches 触点对象集合 TouchList、targetTouches 发生在目标元素上的触点对象集合、changedTouches 状态发生了改变的触点对象集合、ctrlKey、shiftKey、altKey、metaKey、getModifierState
  • SyntheticDragEvent：继承 SyntheticMouseEvent，包含 dataTransfer 拖放期间的传输的数据
  • SyntheticWheelEvent：继承 SyntheticMouseEvent，包含 deltaX 横向滚动量、deltaY 纵向滚动量、deltaZ 指 z 方向滚动量、deltaMode 滚动的单位
  • SyntheticPointerEvent：继承 SyntheticMouseEvent，包含 pointerId 唯一标识、width 指针接触面的宽度、height、pressure 指针施加的压力值、tangentialPressure 切向压力值、tiltX 手写笔与 Y-Z 平面的夹角、tiltY、twist 手写笔围绕自身主轴顺时针旋转的角度、pointerType 触发事件的的设备类型（如鼠标，触控笔，触摸板等）、isPrimary 是否当前设备的主指针。参考 PointerEvent

支撑这些子类的工具函数有：

  • getEventCharCode：获取按键的 character code 数值，处理了兼容性问题。
  • getEventKey：获取按键的 key 物理按键名，内部有调用 String.fromCharCode。
  • getEventModifierState：判断 ctrl、shift、alt、meta 是否被按下。

react 中有以下事件插件：SimpleEventPlugin、BeforeInputEventPlugin、ChangeEventPlugin、EnterLeaveEventPlugin。

事件插件的作用在于当事件触发时，选用匹配的事件插件，通过其 extractEvents 方法获取 SyntheticEvent 等模拟对象实例，并将绑定函数添加到实例属性上，且分发事件。举 TOP_MOUSE_OVER 事件为例，在 SimpleEventPlugin#extractEvents 方法中，有如下处理：

  1. 根据事件类型获取 dispatchConfig（指明绑定事件的方法名为 onMouseOver、onMouseOverCapture）
  2. 获取 SyntheticMouseEvent 类事件对象 event
  3. 通过 accumulateTwoPhaseDispatches 收集组件链路上的实例及相关 onMouseOver、onMouseOverCapture 收集到 event._dispatchListeners、event._dispatchInstances 中。

对于 TOP_MOUSE_OVER 事件，除了 SimpleEventPlugin 插件以外，EnterLeaveEventPlugin 也会对其进行处理，以支持 onMouseEnter、onMouseLeave 等监听。

事件插件通过 EventPluginRegistry 注册，且它会向 npm 包一样发布，以使某个事件类型只能被特定的插件处理。这里说明下 EnterLeaveEventPlugin 的 eventTypes 属性，其键 mouseenter 说明 mouseenter 事件由 EnterLeaveEventPlugin 处理，依赖 TOP_MOUSE_OVER 说明需要对真实 DOM 的 mouseover 事件进行监听。

事件分发

如上文，首先有原生事件 nativeEvent，然后由 SimpleEventPlugin 等插件创建 SyntheticEvent 类实例 event，并为 event 添加相关组件和绑定函数 event._dispatchListeners、event._dispatchInstances。有了 event 及绑定函数后，还需解释绑定函数是怎么执行的。我们可以看到 DOMLegacyEventPluginSystem#runExtractedPluginEventsInBatch，同样使用 TOP_MOUSE_OVER 事件解释如下：

  1. 使用 DOMLegacyEventPluginSystem#extractPluginEvents 获取 event 数组。
  2. 使用 EventBatching#runEventsInBatch，event._dispatchListeners 绑定函数将顺序处理每个 event 对象，直到停止冒泡或所有绑定函数处理完成。

runExtractedPluginEventsInBatch 函数又会被 DOMLegacyEventPluginSystem#handleTopLevel 函数 封装，该函数主要处理多个 react 树嵌套的问题。handleTopLevel 又会被 DOMLegacyEventPluginSystem#dispatchEventForLegacyPluginEventSystem，使用 react-reconciler 包输出的 batchedEventUpdates 方式执行 handleTopLevel。dispatchEventForLegacyPluginEventSystem 最终被封装成 ReactDOMEventListener#dispatchEvent，该函数即所有事件的统一分发接口（其中之一），此处无力解释 dispatchEvent 的实现。

再说事件绑定，ReactDOMEventListener#ReactDOMEventListener 即为真实 DOM 节点绑定 dispatchEvent 等事件分发器，对外透出接口 ReactDOMEventListener#trapBubbledEvent、ReactDOMEventListener#trapCapturedEvent。trapBubbledEvent、trapCapturedEvent 又会被封装成 DOMLegacyEventPluginSystem#legacyListenToTopLevelEvent，它设定了事件是否使用捕获、还是使用冒泡。legacyListenToTopLevelEvent 又会被 DOMLegacyEventPluginSystem#legacyListenToEvent 调用，即对事件插件中的依赖事件（如 onMouseLeave 依赖于 mouseover）进行绑定。最终可以发现，ReactDOMComponent#ensureListeningTo 将事件派发器绑定在了 document 节点上。
